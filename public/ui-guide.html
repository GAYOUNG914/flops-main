Button 컴포넌트 가이드 (Vue 3 + TypeScript)

아래 이미지는 버튼의 주요 상태/스타일(기본, 비활성화, 서치, 링크, 텍스트온리, 라운드, 탭, 컬러, 언더라인 등)을 보여줍니다.

  • 기본: 배경색, 폰트색 지정
  • 비활성화: disabled prop
  • 서치: search prop
  • 링크: link prop
  • 텍스트온리: onlyText14, onlyText16, onlyTextClose14, onlyTextClose16 prop
  • 라운드: round prop
  • 탭: rectangular, tab-round prop
  • 컬러: backgroundColor, fontColor, lineColor 등
  • 언더라인: underlineColor prop
  • 호버: hoverColor, hoverFontColor, hoverLineColor, hoverAction 등

컴포넌트 사용 예시 (Vue Template)

<Button
      text="확인"
      :size="'40'"
      :background-color="'waferblue1'"
      :font-color="'white'"
      :disabled="false"
      :search="false"
      :link="false"
      :only-text-14="false"
      :round="false"
      :hover-action="true"
    />

렌더링 결과 HTML 구조 예시

<button class="btn btn--40 waferblue1">
      <span class="button-text">확인</span>
    </button>
  • .btn: 기본 버튼 클래스
  • .btn--40, .btn--32 등: 사이즈별 클래스
  • .waferblue1, .gray2 등: 배경색 클래스
  • .btn--disabled: 비활성화
  • .btn--search: 서치 아이콘
  • .btn--link: 링크 스타일
  • .btn--only-text-14, .btn--only-text-16, .btn--only-text-close-14, .btn--only-text-close-16: 텍스트온리
  • .btn--round: 라운드
  • .btn--rectangular: 탭/네모
  • .btn--underline: 언더라인 스타일
  • .btn--hover-color, .btn--hover-font-color, .btn--hover-line-color, .btn--hover-action: 호버 효과
  • .button-text: 텍스트 영역
  • .icon-search, .icon-arrow-right, .icon-close: 아이콘

스타일 기준 요약

  • 버튼 높이: 24~48px (size prop)
  • 버튼 border-radius: 4px (round: 100px)
  • 배경색/폰트색: backgroundColor, fontColor prop
  • 언더라인: border-bottom: 1px solid var(--underline-color, var(--color-gray3)) (underlineColor prop)
  • 비활성화: background: var(--color-gray4), color: var(--color-white)
  • 서치/링크/텍스트온리/라운드/탭 등 다양한 변형 지원
  • 호버 효과: hoverColor, hoverFontColor, hoverLineColor, hoverAction

참고

  • props로 다양한 스타일/상태 지원
  • 컴포넌트는 scoped style로 작성됨
  • 아이콘은 /icon 경로의 SVG 파일 사용

TextInput 컴포넌트 가이드 (Vue 3 + TypeScript)

아래 이미지는 텍스트 인풋의 주요 상태(기본, 포커스, 입력됨, 완료, 비활성화, 에러, 읽기전용, 콤보, 검색)를 보여줍니다.

  • 기본: 입력 전 상태
  • 포커스: 입력창에 포커스된 상태
  • 입력됨: 값이 입력된 상태
  • 완료: completed 상태
  • 비활성화: disabled 상태
  • 에러: error 상태
  • 읽기전용: readonly 상태
  • 콤보: combo 아이템이 있을 때
  • 검색: search prop이 true일 때

컴포넌트 사용 예시 (Vue Template)

<TextInput
  v-model="inputValue"
  :placeholder="'이름을 입력하세요'"
  :count="20"
  :error="errorMessage"
  :readonly="false"
  :disabled="false"
  :combo="comboList"
  :search="true"
/>

렌더링 결과 HTML 구조 예시

<div class="text-input pressed has-count">
    <div class="input-wrapper">
      <input type="text" class="" placeholder="이름을 입력하세요" value="홍길동">
      <button class="clear-btn" type="button">
        <span aria-hidden="true">
          <img src="/portal-ui/icon/icon-clear.svg" alt="close">
        </span>
      </button>
      <div class="icon-search">
        <img src="/portal-ui/icon/icon-input-search.svg" alt="search">
      </div>
      <div class="combo-list">
        <span class="combo-item">태그1 <span class="combo-remove">✕</span></span>
        <span class="combo-item">태그2 <span class="combo-remove">✕</span></span>
      </div>
    </div>
    <div class="messages">
      <div class="subtext">서브텍스트 예시</div>
      <div class="count">
        <span class="count-text">3</span>/<span class="count-text">20</span>
      </div>
    </div>
  </div>
  • .text-input: 전체 wrapper
  • .input-wrapper: 인풋 컨테이너
  • input: 실제 입력창
  • .clear-btn: 입력값 삭제 버튼
  • .icon-search: 검색 아이콘
  • .combo-list: 콤보 아이템 리스트
  • .combo-item: 콤보 아이템
  • .combo-remove: 콤보 아이템 삭제
  • .error-msg: 에러 메시지
  • .messages: 하단 메시지 영역
  • .subtext: 서브텍스트
  • .count: 글자수 표시
  • .count-text: 글자수 텍스트
  • 상태별 클래스: pressed, focus, completed, is-disabled, readonly, error, is-search, has-count, has-error, has-subtext, is-combo

스타일 기준 요약

  • 입력창 크기: width: 320px; height: 32px;
  • 입력창 패딩: 0 16px
  • 입력창 border-radius: 4px
  • 입력창 기본 border: 1px solid var(--color-gray4)
  • 포커스/입력됨: border-color: #03091D
  • 완료: border-color: var(--color-gray4)
  • 비활성화: color: var(--color-gray4)
  • 읽기전용: color: var(--color-gray2)
  • 에러: border-color: var(--color-error1)
  • placeholder 색상: #666 (포커스/호버 시 #03091D)
  • 콤보 아이템: .combo-list 하위에 표시
  • 검색 아이콘: .icon-search로 우측에 표시
  • 입력값 삭제 버튼: .clear-btn로 우측에 표시
  • 글자수: .count로 우측 하단 표시
  • 에러 메시지: .error-msg로 하단 표시

참고

  • 글자수 제한이 있을 경우 count prop으로 지정
  • 에러 메시지는 error prop으로 표시
  • 완료/비활성/읽기전용 상태에서는 입력 불가
  • 콤보/검색 등 다양한 상태 지원
  • 컴포넌트는 scoped style로 작성됨

Textarea 컴포넌트 가이드 (Vue 3 + TypeScript)

아래 이미지는 텍스트에어리어의 주요 상태(기본, 포커스, 입력됨, 완료, 비활성화, 에러, 읽기전용)를 보여줍니다.

  • 기본: 입력 전 상태
  • 포커스: 입력창에 포커스된 상태
  • 입력됨: 값이 입력된 상태
  • 완료: completed 상태
  • 비활성화: disabled 상태
  • 에러: error 상태
  • 읽기전용: readOnly 상태

컴포넌트 사용 예시 (Vue Template)

<Textarea

  • .textarea-container: 텍스트에어리어 컨테이너

  • .textarea-input: textarea 입력창
  • .textarea-bottom: 하단 영역(에러 메시지, 글자수)
  • .error-message: 에러 메시지
  • .character-count: 글자수 표시
  • .character-count-number: 현재 글자수

스타일 기준 요약

  • 입력창 패딩: 14px 18px
  • 글자수 표시 위치: 우측 하단
  • 에러 메시지 위치: 좌측 하단
  • 입력창 border-radius: 4px
  • 입력창 기본 border: 1px solid var(--color-gray4)
  • 포커스/입력됨: border-color: var(--color-black)
  • 완료: border-color: var(--color-gray4)
  • 비활성화: border-color: var(--color-gray4), color: var(--color-gray3)
  • 에러: border-color: var(--color-error1), 글자수/에러 메시지 color: var(--color-error1)
  • 읽기전용: border-color: var(--color-gray3), color: var(--color-gray2)
  • placeholder 색상: var(--color-gray3) (포커스/호버 시 var(--color-black))

참고

  • 글자수 제한이 있을 경우 maxLength로 지정
  • 에러 메시지는 error와 errorMessage 조합으로 표시
  • 완료/비활성/읽기전용 상태에서는 입력 불가
  • 컴포넌트는 scoped style로 작성됨

Radio Button 컴포넌트 가이드 (Vue 3 + TypeScript)

아래 이미지는 라디오 버튼의 네 가지 상태를 보여줍니다.

  • Normal Disabled: 비활성화 상태 (선택 안 됨)
  • Selected Disabled: 비활성화 상태 (선택됨)
  • Unchecked: 기본 선택되지 않은 상태
  • Checked: 선택된 상태

컴포넌트 사용 예시 (Vue Template)

<Radio
  v-model="selectedOption"
  :value="'option1'"
  label="옵션 1"
  name="exampleRadio"
/>

렌더링 결과 HTML 구조 예시

<label class="radio checked">
  <input type="radio" name="exampleRadio" value="option1" checked />
  <span class="radio-icon"></span>
  <span class="radio-label b3_ko_s">옵션 1</span>
</label>
  • .radio: 기본 wrapper
  • .checked: 선택된 상태
  • .disabled: 비활성화 상태일 경우 추가됨
  • .radio-icon: 아이콘 영역 (배경이미지로 상태 구분)
  • .radio-label: 텍스트 라벨 영역

스타일 기준 요약

  • 아이콘 크기: 20px × 20px
  • 라벨 간 간격: 8px
  • 배경 이미지:
    • 기본: radio-unchecked.svg
    • 선택됨: radio-checked.svg
    • 비활성화: radio-unchecked-disabled.svg
    • 선택+비활성화: radio-checked-disabled.svg
  • 선택 상태 시: font-weight: 700
  • 비활성 라벨 색상: var(--color-gray3)

참고

본 컴포넌트는 scoped style로 작성되어 있으며, 아이콘은 /icon 경로의 SVG 파일을 사용합니다.

Dropdown 컴포넌트 가이드 (Vue 3 + TypeScript)

아래 이미지는 드롭다운의 주요 상태(기본, 포커스, 입력됨, 완료, 비활성화, 에러, 읽기전용, 다중선택, 라디오스타일, 검색)를 보여줍니다.

  • 기본: 아무것도 선택되지 않은 상태
  • 포커스: 드롭다운에 포커스된 상태
  • 입력됨: 값이 선택된 상태(pressed)
  • 완료: completed 상태
  • 비활성화: disabled 상태
  • 에러: error 상태
  • 읽기전용: readonly 상태
  • 다중선택: multiple prop이 true일 때
  • 라디오스타일: radioStyle prop이 true일 때
  • 검색: searchable prop이 true일 때

컴포넌트 사용 예시 (Vue Template)

<CustomDropdown
  v-model="selectedValue"
  :options="dropdownOptions"
  placeholder="옵션을 선택하세요"
  :multiple="false"
  :disabled="false"
  :readonly="false"
  :variant="'default'"
  :error-message="errorMessage"
  :show-clear="true"
  :searchable="true"
  search-placeholder="검색어 입력"
  :show-check-icon="true"
  :radio-style="false"
/>
